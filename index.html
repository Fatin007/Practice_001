B - Picnic Chaos !!

This problem has a number of solutios. First calculate sum from 1 to n (1+2+...+n). Then take input and substract it from sum. after all step remaining sum will be the missing ID number.


B - Picnic Chaos!!

Code:

                          
#include <stdio.h>
int main()
{
   int n, x, sum = 0;
   scanf("%d", &n);
   for (int i = 1; i <= n; i++)
   {
      sum += i;
   }
   for (int i = 0; i < n-1; i++)
   {
      scanf("%d", &x);
      sum -= x;
   }
   printf("%d", sum);
}

Explaination:
The first loop calculates the sum of numbers from 1 to 'n.' The second loop subtracts the input numbers, and the remaining sum is the missing number.

C - !Easy Math?

You've to write a simple program to calculate factorial for T testcases. Here edge case if factorial of zero!


C - !Easy Math?

Code:

                          
#include <stdio.h>
int main()
{
   int t;
   scanf("%d", &t);
   while (t--)
   {
      int n;
      long long fact = 1;
      scanf("%d", &n);
      if (n == 0)
      {
         printf("1\n");
         continue;
      }
      for (int i = 1; i <= n; i++)
      {
         fact *= i;
      }
      printf("%lld\n", fact);
   }
}

F - Reverse

Here's another simple program that reverses an integer, and you've to write it for multiple test cases.


F - Reverse

Code:

                            
#include <stdio.h>
int main()
{
   int t;
   scanf("%d", &t);
   while (t--)
   {
      int n, rem, rev = 0;
      scanf("%d", &n);
      while (n)
      {
         rem = n % 10;
         rev = rev * 10 + rem;
         n /= 10;
      }
      printf("%d\n", rev);
   }
}

H - Noshto lift:"'

This problem has many solutions. Here is a mathematical approach to solve it.
Let the last digit be 'last.' Then, our character pressed each digit before the last digit exactly 10 times (1+2+3+4). The number of times he pressed the last digit depends on the length of the number. So, the final answer is
(last-1)*10+length*(length+1)/2.


H - Noshto lift:"'

Code:

                            
#include <stdio.h>
#include <math.h>
int main()
{
   int t;
   scanf("%d", &t);
   while (t--)
   {
      int n, sum = 0, last, length;
      scanf("%d", &n);
      last = n % 10;
      length = 1 + log10(n);
      sum = (last - 1) * 10 + (length * (length + 1) / 2);
      printf("%d\n", sum);
   }
}

Explaination:
Here, 1+log10(n) is the shortcut formula to get length of number of digits of a number.

I - The marathon

We can rephrase the problem to count the numbers from 'j,' 'a,' and 't' that are larger than 's.' One way to accomplish this is by maintaining a variable that increments each time we use an 'if' statement to check if a number is greater than 's'.


I - The marathon

Code:

                            
#include <stdio.h>
int main()
{
   int t;
   scanf("%d", &t);
   while (t--)
   {
      int s, j, a, t, count = 0;
      scanf("%d%d%d%d", &s, &j, &a, &t);
      if (s < j) count++;
      if (s < a) count++;
      if (s < t) count++;
      printf("%d\n", count);
   }
}

J - A Variety of Operations

If you closely observe the problem, you will identify four conditions:

  1. If 'c - d' is even, the output will be -1.
  2. If 'c' and 'd' are both equal to 0, the answer is 0.
  3. If 'c' is equal to 'd,' the answer is 1.
  4. Otherwise, the answer is 2.


J - A Variety of Operations

Code:

                            
#include <stdio.h>
int main()
{
   int t;
   scanf("%d", &t);
   while (t--)
   {
      int c, d;
      scanf("%d%d", &c, &d);
      if ((c - d) % 2 == 1)
         printf("-1\n");
      else if (c == 0 && d == 0)
         printf("0\n");
      else if (c == d)
         printf("1\n");
      else
         printf("2\n");
   }
}
